Gateway keystore encryption (#17):
  GATEWAY_KEYSTORE_KEY is REQUIRED for gateway boot. The chart wires it via
  the gateway-credentials Secret. Provision per backend:
    k8s-native       ‚Äî pass via: --set gateway.secrets.values.gatewayKeystoreKey="$(openssl rand -base64 32)"
    vault/aws/azure  ‚Äî populate property `gatewayKeystoreKey` in the remote store
  Gateway will refuse to boot without it (fail-closed). Format: base64 of 32
  raw bytes (AES-256). Rotation requires re-keying the on-disk keystore;
  re-running install with a different value invalidates every encrypted record.

=======================================================
  ‚öÝ  DATA DESTRUCTION ON UNINSTALL ‚Äî READ BEFORE `helm uninstall`
=======================================================

The compliance databases (audit, id-bridge, veil-witness) hold your
REGULATOR-CITED EVIDENCE: the immutable audit trail (AI Act Art. 12 /
GDPR Art. 30), the re-linkage tokens (GDPR DSAR / erasure), and the
signed witness certificates. This data is NOT recoverable once gone.

  ‚Ä¢ The PVCs for these three databases carry `helm.sh/resource-policy: keep`,
    so `helm uninstall` does NOT delete them. The data SURVIVES an uninstall
    and is re-bound on the next install into the same namespaces.
  ‚Ä¢ To DELETE the data on purpose you must remove the PVCs explicitly:
      kubectl delete pvc audit-postgresql-data       -n dsa-audit
      kubectl delete pvc id-bridge-postgresql-data    -n dsa-bridge
      kubectl delete pvc veil-witness-postgresql-data  -n dsa-witness
  ‚Ä¢ BEFORE any uninstall/reinstall or PVC deletion: take a backup.
    See OPS.md ¬ß Backups for the pg_dump / pg_restore recipes (Compose + Helm)
    and the restore-validation step.
