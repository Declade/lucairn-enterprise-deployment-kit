Lucairn Enterprise Dashboard

1) The dashboard is reachable via the Service:
   {{ include "dashboard.fullname" . }}.{{ .Values.namespace }}.svc.cluster.local:{{ .Values.service.port }}

   For local access:
     kubectl -n {{ .Values.namespace }} port-forward svc/{{ include "dashboard.fullname" . }} {{ .Values.service.port }}:{{ .Values.service.port }}

   Then open: https://localhost:{{ .Values.service.port }}/login

   The dashboard expects a TLS-terminating reverse proxy in front. The pod
   does NOT terminate TLS itself; your Ingress or front-proxy must do so
   before traffic reaches the Service.

2) Retrieve the bootstrap admin credentials:
   - Email (operator-overridable via Helm value bootstrapAdmin.email):
       {{ .Values.bootstrapAdmin.email }}
   - Password (random per install; retrieve once and rotate per OPS.md):
       kubectl -n {{ .Values.namespace }} get secret \
         {{ include "dashboard.bootstrapAdminSecretName" . }} \
         -o jsonpath='{.data.password}' | base64 -d

3) Rotate the password (OPS.md procedure):
   kubectl -n {{ .Values.namespace }} create secret generic {{ include "dashboard.bootstrapAdminSecretName" . }} \
     --from-literal=password='<new-password>' \
     --from-literal=session-secret='<new-session-secret>' \
     --dry-run=client -o yaml | kubectl apply -f -
   kubectl -n {{ .Values.namespace }} rollout restart deploy/{{ include "dashboard.fullname" . }}

{{- if .Values.oidc.enabled }}

4) OIDC SSO is ENABLED.
   - Issuer URL: {{ .Values.oidc.issuerURL }}
   - Client ID: {{ .Values.oidc.clientID }}
   - Admin group: {{ .Values.oidc.adminGroup }}
   - Viewer group: {{ .Values.oidc.viewerGroup }}
   - Client-secret Secret: {{ .Values.oidc.clientSecretRef.name }} (key: {{ .Values.oidc.clientSecretRef.key | default "client-secret" }})

   The "Sign in with SSO" button appears on the /login page alongside the
   local form. Local-admin sign-in continues to work for bootstrap and
   IdP-outage scenarios.

   Rotate the OIDC client_secret (OPS.md procedure):
   kubectl -n {{ .Values.namespace }} create secret generic {{ .Values.oidc.clientSecretRef.name }} \
     --from-literal={{ .Values.oidc.clientSecretRef.key | default "client-secret" }}='<new-secret>' \
     --dry-run=client -o yaml | kubectl apply -f -
   kubectl -n {{ .Values.namespace }} rollout restart deploy/{{ include "dashboard.fullname" . }}
{{- else }}

4) OIDC SSO is DISABLED. To enable: set dashboard.oidc.enabled=true plus the
   required fields (issuerURL, clientID, clientSecretRef.name, adminGroup,
   viewerGroup, publicURL OR callbackURL) and `helm upgrade`. The login
   surface will then render a "Sign in with SSO" button.
{{- end }}
